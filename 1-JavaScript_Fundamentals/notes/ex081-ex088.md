## Exercise 081

Statements are typically separated by semicolons. However, a line break can often be treated as a statement separator through a mechanism called *Automatic Semicolon Insertion (ASI)*.

Because ASI does not always work as expected, it is generally recommended to place a semicolon after each statement.

Semicolons are not required after blocks enclosed in `{...}` or after syntactic constructs that contain such blocks, such as loops.

Adding an extra semicolon is not considered an error. It simply represents an empty statement.

## Exercise 082

To enable all modern JavaScript features, scripts should begin with `"use strict"`.

This directive must appear at the top of a script or at the beginning of a function body.

Without strict mode, code will still run, but some features may behave according to older compatibility rules.

Certain language features, such as classes, automatically enable strict mode.

## Exercise 083

Variables can be declared using:

* `let`
* `const` - declares a constant value that cannot be reassigned
* `var` - the older style of variable declaration

A variable name may contain:

* Letters
* Digits (except as the first character)
* The characters `$` and `_`

Non-Latin alphabets and even hieroglyphic characters are technically allowed, although they are rarely used in practice.

JavaScript is a dynamically typed language, meaning that a variable can store values of different types over time.

JavaScript provides eight data types:

* `number` - for both integers and floating-point numbers
* `bigint` - for arbitrarily large integers
* `string` - for text values
* `boolean` - for logical values: `true` and `false`
* `null` - represents an empty or non-existent value
* `undefined` - represents a value that has not been assigned
* `object` - for complex data structures
* `symbol` - for unique identifiers

The `typeof` operator returns the type of a value.

There are two well-known exceptions:

* `typeof null` returns `"object"`, which is a historical mistake in the language.
* `typeof function() {}` returns `"function"`. Technically, functions are objects, but JavaScript treats them as a special case.

## Exercise 084

Since we are using a web browser as our environment, the basic user interface functions are:

* `prompt(question, [default])` - asks a question and returns the user's input, or `null` if the user clicks Cancel.
* `confirm(question)` - asks a question and allows the user to choose between OK and Cancel. The result is returned as `true` or `false`.
* `alert(message)` - displays a message.

All of these functions are *modal*.

They pause script execution and prevent the user from interacting with the page until a response is provided.

## Exercise 085

JavaScript supports the following operators:

### Arithmetic Operators

The standard arithmetic operators are:

* `+`
* `-`
* `*`
* `/`

Additional operators include:

* `%` for the remainder
* `**` for exponentiation

The binary `+` operator performs addition, but it also concatenates strings. If either operand is a string, the other operand is converted to a string as well.

### Assignment Operators

Examples include:

* Simple assignment: `a = b`
* Compound assignment: `a *= 2`

### Bitwise Operators

Bitwise operators work with 32-bit integers at the binary level.

### Conditional Operator

The only operator that accepts three operands:

```js
cond ? resultA : resultB
```

If `cond` is true, the result is `resultA`; otherwise, the result is `resultB`.

### Logical Operators

The logical operators `&&` and `||` perform short-circuit evaluation and return the value where evaluation stops.

The logical NOT operator `!` converts its operand to a boolean and returns the opposite value.

### Nullish Coalescing Operator

The `??` operator provides a way to select the first defined value from a list of expressions.

The result of:

```js
a ?? b
```

is `a`, unless `a` is `null` or `undefined`, in which case `b` is returned.

### Comparison Operators

The equality operator `==` performs type conversion when comparing values of different types.

An exception exists for `null` and `undefined`, which are equal to each other and to no other value.

The strict equality operator `===` does not perform type conversion. Different types always produce a result of `false`.

Greater-than and less-than comparisons compare strings character by character. Other values are generally converted to numbers.

### Other Operators

JavaScript also includes additional operators, such as the comma operator.

## Exercise 086

Three loop types were covered:

* `while`
* `do...while`
* `for`

Variables declared inside a `for (let ...)` loop are only visible within that loop.

Alternatively, an existing variable can be reused by omitting `let`.

## Exercise 088

We covered three ways to create functions in JavaScript:

1. **Function Declaration** - a function defined as a separate statement in the main code flow.
2. **Function Expression** - a function created within an expression.
3. **Arrow Function** - a shorter syntax for creating functions.

Functions can have local variables, which include:

* Variables declared inside the function body.
* Parameters declared in the function definition.

These variables are only accessible within the function itself.

Parameters may also have default values:

```js
function sum(a = 1, b = 2) {
  ...
}
```

Functions always return a value.

If no `return` statement is provided, the result is `undefined`.
