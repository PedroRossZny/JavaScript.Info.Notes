## Exercise 028

An operand is the value to which an operator is applied. For example, in `5 * 2`, the operands are `5` and `2`, while the operator is `*`. An operator is considered *unary* when it has a single operand, such as the unary `-` used for negation. An operator is *binary* when it has two operands. The minus sign also exists in its binary form, where it is used for subtraction. As shown in **ex028**.

## Exercise 029

The following mathematical operations are supported: Addition `+`, Subtraction `-`, Multiplication `*`, Division `/`, Remainder `%`, and Exponentiation `**`. The `%` operator returns the remainder of a division between two operands. The `**` operator raises the first operand to the power of the second. It can also be used to calculate roots by raising a number to fractional powers, such as `1/2`. As shown in **ex029**.

## Exercise 030

In addition to adding numbers, the `+` operator can also concatenate strings. When an operation involves a number and a string, the result is converted into a string. In more complex expressions with multiple operands, operations are evaluated from left to right. Once a string operand is encountered, the result accumulated so far is converted into a string and concatenation continues. The `+` operator is the only operator that supports strings in this way. Other arithmetic operators work only with numbers and always convert their operands to numeric values. As shown in **ex030**.

## Exercise 032

Operations follow a precedence order. When operators have the same precedence, evaluation proceeds from left to right. Parentheses override the default precedence rules. A simplified precedence order is: unary plus `+`, unary negation `-`, exponentiation `**`, multiplication `*`, division `/`, addition `+`, subtraction `-`, and assignment `=`.

Using `=`, it is possible to perform chained assignments, which are evaluated from right to left. However, for better readability, it is usually preferable to split them across multiple lines. As shown in **ex032**.

These operators have the same precedence as a regular assignment, meaning they are executed after most other calculations. As shown in **ex033**.

## Exercise 034

Increasing or decreasing a number by one is a very common operation, which is why JavaScript provides specific operators for it. The `++` operator increases a variable by `1`, while `--` decreases it by `1`.

These operators work only with variables. Attempting to use them directly on numeric literals results in an error. They can be placed either before or after a variable. Both forms perform the increment or decrement, but they return different values.

The *prefix* form, such as `++counter`, increments the variable and returns the new value. The *postfix* form, such as `counter++`, also increments the variable but returns the previous value. If the result of the increment or decrement is not used, there is no practical difference between the two forms.

The `++` and `--` operators can also be used within expressions, and their precedence is higher than that of most other operators. As shown in **ex034**.

## Exercise 035

The comma operator `,` is one of the least commonly used operators. It is sometimes employed to write shorter code. This operator allows multiple expressions to be evaluated by separating them with commas. All expressions are evaluated, but only the result of the last expression is returned.

It is often used in more complex constructs to perform multiple actions within a single line of code. However, it should be used carefully, as it generally does not improve code readability. As shown in **ex035**.
