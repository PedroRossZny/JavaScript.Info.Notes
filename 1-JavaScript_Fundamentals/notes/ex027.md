# JavaScript Type Conversions

## Exercise 027

Function operators usually convert values to the correct type automatically. For example, `alert` converts any value into a *string* in order to display it. Mathematical operations also convert values into numbers when necessary. However, there are situations where we need to explicitly convert a value to the expected type.

The function used to convert a value into a string is `String(value)`. Its behavior is generally straightforward: `false` becomes `"false"`, `null` becomes `"null"`, and so on.

Numeric conversion in functions and expressions often happens automatically, but we can use `Number(value)` to explicitly convert a value into a number. If the string is not a valid number, the result of the conversion will be `NaN`.

Some common conversion results are:

* `undefined` → `NaN`
* `null` → `0`
* `true` and `false` → `1` and `0`

For strings, leading and trailing whitespace characters, including spaces, `\t`, `\n`, and others, are removed. If the remaining string is empty, the result is `0`. Otherwise, JavaScript attempts to extract a number from the string. If it cannot, the result is `NaN`.

For Boolean conversion, `Boolean(value)`, the rules are intuitive. Values that represent "empty" are converted to `false`, such as:

* `0`
* an empty string (`""`)
* `null`
* `undefined`
* `NaN`

All other values are converted to `true`.

In PHP, the string `"0"` is treated as `false`, but in JavaScript any non-empty string is always `true`.
