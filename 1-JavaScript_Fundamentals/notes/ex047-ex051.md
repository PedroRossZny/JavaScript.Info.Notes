# JavaScript Logical Operators

## Exercise 047

The logical *OR* operator is represented by `||`.

When used with boolean values, it returns `true` if at least one operand is `true`. Otherwise, it returns `false`.

In other words, the result is `false` only when all operands are `false`.

If an operand is not a boolean value, it will be converted to a boolean during the evaluation process.

Most of the time, the `||` operator is used inside `if` statements to test whether at least one of several conditions is true.

Any number of conditions can be combined using `||`. As shown in **ex047**.

## Exercise 048

The `||` operator finds the first truthy value.

It works as follows:

1. Operands are evaluated from left to right.
2. Each operand is converted to a boolean value.
3. If the result is `true`, the operation stops and returns the original value of that operand.
4. If all operands are falsy, the last operand is returned.

This behavior can be used in interesting ways. For example, when evaluating string variables, an empty string is considered falsy and is skipped, while the first non-empty string is returned.

Keep in mind that if all operands are falsy, the last value is returned.

The `||` operator also uses *short-circuit evaluation*. Once the first truthy value is found, the remaining operands are not evaluated.

## Exercise 049

The logical *AND* operator is represented by `&&`.

This operator returns `true` only when all operands are `true`. Otherwise, it returns `false`.

Like `||`, any value can be used as an operand. During evaluation, operands are converted to boolean values, but the operator ultimately returns one of the original operands.

## Exercise 050

The `&&` operator finds the first falsy value.

It works as follows:

1. Operands are evaluated from left to right.
2. Each operand is converted to a boolean value.
3. If the result is `false`, the operation stops and returns the original value of that operand.
4. If all operands are truthy, the last operand is returned.

This behavior is very similar to the *OR* operator. The difference is that `&&` returns the first falsy value, while `||` returns the first truthy value.

Multiple operands can be chained together.

The precedence of `&&` is higher than the precedence of `||`. This means that `&&` expressions are evaluated before `||` expressions.

Sometimes `&&` and `||` are used as shorter alternatives to `if` statements. Although this may reduce the amount of code, it can also make the code harder to read.

As a general rule, use `if` when you need conditional branching, use `&&` when you mean *AND*, and use `||` when you mean *OR*. As shown in **ex050**.

## Exercise 051

The logical *NOT* operator is represented by `!`.

It converts its operand to a boolean value and returns the opposite result.

There is also a double version, `!!`, which is commonly used to convert a value to a boolean.

It works as follows:

1. The first `!` converts the value to a boolean and inverts it.
2. The second `!` inverts the result again.

The final result is simply the boolean representation of the original value.

The same conversion can be performed more clearly using the built-in `Boolean()` function.

The `!` operator has the highest precedence among all logical operators. As shown in **ex051**.
