# JavaScript Nullish Coalescing Operator `??`

## Exercise 052

The `??` operator is called the *nullish coalescing operator*.

It provides a default value when the left operand is `null` or `undefined`.

The result of `a ?? b` is:

* If `a` is defined (not `null` and not `undefined`), return `a`.
* Otherwise, return `b`.

This behavior can be reproduced using operators that existed before `??`, since the nullish coalescing operator was introduced later in JavaScript.

The operator can also be chained across multiple values. In that case, it returns the first value that is not `null` or `undefined`, evaluating from left to right.

The `??` operator is very similar to `||`, which was traditionally used for this purpose. However, `??` was introduced to avoid problems that can occur when using `||`.

The `||` operator treats values such as `false`, `0`, and `""` as falsy and may incorrectly replace them with a default value. The `??` operator is more restrictive and only considers `null` and `undefined` to be missing values.

## Exercise 053

The `??` operator has the same precedence as `||`, which is `3` in the *MDN operator precedence table*.

Because of this, `??` is evaluated before assignment operators such as `=`, and before the conditional operator `? :`, but after most other operators.

To avoid ambiguity and improve readability, it is generally recommended to use parentheses when `??` appears in more complex expressions.

## Exercise 054

For safety reasons, JavaScript does not allow `??` to be combined directly with `||` or `&&`.

To use these operators together, the intended precedence must be made explicit with parentheses.

For example:

```js
(a ?? b) || c
```

or

```js
a ?? (b && c)
```

Always use parentheses when combining `??` with `||` or `&&` to avoid syntax errors and make the code easier to understand.
