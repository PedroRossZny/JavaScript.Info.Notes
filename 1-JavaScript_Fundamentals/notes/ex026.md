# JavaScript Interaction: alert, prompt, confirm

## Exercise 026

We have already seen `alert`, which displays a message and waits for the user to press **OK**. The message window is called a *modal window*, meaning that the visitor cannot interact with the rest of the page until they have interacted with the window.

The `prompt` function accepts two arguments: `title`, the text that will be displayed, and `default`, an optional second parameter that contains an initial value for the input field. The `[...]` notation indicates that the parameter is optional.

This function returns the entered text as its result. However, if the user clicks **Cancel** or presses the **ESC** key, `null` is returned instead.

The second parameter is optional, but in Internet Explorer, if no value is provided, the text `undefined` will appear in the input field.

The `confirm` function displays a window containing a question and two buttons: **OK** and **Cancel**. It returns `true` when **OK** is pressed and `false` when **Cancel** is pressed. It also returns `false` if the **ESC** key is pressed.
