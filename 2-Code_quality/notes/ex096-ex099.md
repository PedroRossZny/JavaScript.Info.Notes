# JavaScript Comments

## Exercise 096

Comments can be either single-line comments using `//` or multi-line comments using `/* ... */`.

Comments are typically used to describe how and why code works.

Many beginners use comments to explain exactly what the code is doing. However, in well-written code, the number of explanatory comments should be minimal because the code itself should be easy to understand.

A common guideline is:

> "If the code is so confusing that it requires a comment, it might be better to rewrite it."

## Exercise 097

In many cases, it is beneficial to replace a block of code with a function.

A well-named function often serves as a comment by itself because it is *self-descriptive*.

This style of programming is commonly referred to as *self-documenting code*.

## Exercise 098

When working with a large block of code, it can often be improved through refactoring.

Breaking the code into smaller functions allows the function names themselves to explain what is happening.

As a result, fewer comments are needed, and the structure of the code becomes clearer.

It becomes easier to understand:

* What each function does.
* What parameters it receives.
* What value it returns.

Of course, comments cannot be avoided completely.

Some algorithms are inherently complex, and certain optimizations may require clever or non-obvious solutions.

In general, however, we should minimize the use of comments whenever possible and strive to write code that is simple, clear, and self-explanatory.

## Exercise 099

Good comments are comments that describe the overall structure of the code.

They provide a high-level overview of the components, explain how they interact, and describe the control flow in different situations.

In other words, they present the "big picture" of the program.

There is also a special syntax called *JSDoc* that can be used to document functions, including:

* Their purpose.
* Their parameters.
* Their return values.

These comments allow developers to understand and use a function correctly without having to inspect its implementation.

Tools such as *JSDoc 3* can generate HTML documentation directly from these comments.

More information can be found at:

https://jsdoc.app/

Comments can also be used to explain *why* a particular solution was chosen.

This is often more valuable than explaining *what* the code does, since the code itself should already make that clear.

Another useful purpose for comments is documenting subtle behavior or non-obvious implementation details.

If a piece of code contains behavior that is difficult to understand, surprising, or counterintuitive, it is usually worth adding a comment to explain it.
