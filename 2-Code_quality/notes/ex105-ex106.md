# JavaScript Polyfills and transpilers

The JavaScript language is constantly evolving, and as programmers, the more features that make our lives easier, the better. On the other hand, how can we make our modern code work in older JavaScript engines that do not yet understand the latest features? With *transpilers* and *polyfills*.

## Exercise 105

A *transpiler* is specialized software that translates source code into another version of source code. It analyzes modern code and rewrites it using older syntax constructs. Developers usually run the transpiler on their own computers and then deploy the transpiled code to the server. [Babel](https://babeljs.io/) is one of the most important transpilers available. Modern project build tools such as [webpack](https://webpack.js.org/) can automatically run a transpiler whenever the code changes, making it very easy to integrate into the development workflow. ## Exercise 106

New language features may also include built-in functions. In that case, there is no need to transpile anything; we only need to provide the missing function. A script that adds or updates missing language features is called a *polyfill*. An interesting *polyfill* library is [core-js](https://github.com/zloirock/core-js), which supports a wide range of features and allows you to include only the ones you need. As shown in **ex106**.
