## Exercise 155

It is also possible to create a multi-line string using single or double quotes by using
the special character `\n` for line breaks. There are other less common special characters:

- `\n` — new line.
- `\r` — on Windows text files, a combination of `\r\n` represents a line break, while on
non-Windows systems only `\n` is used. This is due to historical reasons; most Windows
software also understands `\n`.
- `\'`, `\"`, `` \` `` — quotes.
- `\\` — backslash.
- `\t` — tab.
- `\b`, `\f`, `\v` — Backspace, Form Feed, Vertical Tab — mentioned for completeness;
these are old and no longer used today.

As already noted, the backslash is what we use to introduce special characters, hence it
is also called the *escape character*. Escaped quotes such as `\'`, `\"`, and `` \` `` are
used to insert a quote inside a string that uses the same type of quote. In some cases
they are not needed — if we start a string with backticks or double quotes, we can freely
place single quotes inside the text, as long as it is different from the opening quote, as
shown in **ex155**.

## Exercise 160

There are many ways to search for a substring within a string. The first is the method
`str.indexOf(substr, pos)`, which returns the starting position of the substring if found,
or `-1` if not. The second argument is optional and controls where the search begins. To
find all occurrences in a string we can run `indexOf` in a loop, with each call starting
from the position after the previous match.

There is also `str.lastIndexOf(substr, position)`, which works in the opposite direction.

When using `indexOf` in an `if` check, we must not forget that if the method returns `0`
the `if` will treat it as `false`. We should always check whether the result is different
from `-1`. A more modern alternative that avoids this issue is `str.includes(substr, pos)`,
which returns `true` or `false` depending on whether the substring was found. It also
accepts an optional starting position as a second argument.

Other useful methods are `str.startsWith` and `str.endsWith`, which return `true` or
`false` depending on whether the string starts or ends with the given substring, as shown
in **ex160**.

## Exercise 161

There are 3 methods in JavaScript to extract a substring:

- `str.slice(start [, end])` — returns the part of the string from `start` up to but not
including `end`. If no second argument is provided, it goes until the end of the string.
Negative values are allowed and count from the end of the string.
- `str.substring(start [, end])` — works like `slice`, but allows `start` to be greater
than `end`, in which case it swaps the two values. Negative numbers are treated as `0`.
- `str.substr(start [, length])` — returns the part of the string starting at `start`
with the given number of characters. The first argument can be negative, counting from
the end. This method is only officially supported in browser-hosted JavaScript engines and
its use is not recommended, though in practice it works everywhere.

Summary:

| Method | Selects | Negatives |
|--------|---------|-----------|
| `slice(start, end)` | From `start` to `end` (excluding `end`) | Allowed |
| `substring(start, end)` | Between `start` and `end` (excluding `end`) | Treated as `0` |
| `substr(start, length)` | From `start`, gets `length` characters | Allowed for `start` |

`slice` is the most widely used and recommended method due to its flexibility, as shown
in **ex161**.

## Exercise 162

When comparing strings in JavaScript, lowercase letters are always greater than uppercase
ones. Letters with diacritical marks are also out of order, which can lead to unexpected
results, especially when working with names from different countries.

The method `str.codePointAt(pos)` returns the numeric code of the character at a given
position, and `String.fromCodePoint(code)` creates a character from its code. Using these
methods helps us understand how characters are compared — by their numeric code, where a
higher code means a greater character.

For a more linguistically correct comparison, we have `str.localeCompare(str2)`, which
returns `-1` if `str` is less than `str2`, `0` if they are equal, and `1` if `str` is
greater. This comparison is adapted to the appropriate language, as shown in **ex162**.
