# Iterables

## Exercise 186

*Iterable objects* are a generalization of arrays, allowing us to make any object usable
in a `for..of` loop. If an object is not technically an array but represents a list of
something and we want to iterate over it like an array, we need to add a method to the
object called `Symbol.iterator`. When `for..of` starts, it calls this method once — if it
is not found, an error is thrown. The method must return an iterator, which is an object
with a `next` method. From that point on, the loop works only with the returned object,
calling `next` each time it needs the next value. The result of `next()` must have the
form `{done: Boolean, value: any}` — if `done` is `true` the loop ends, otherwise
`value` is the next value.

We can observe the separation of concerns in iterable objects: the object itself does not
have a `next()` method — instead, an iterator is created separately, and `next()` generates
values for the iteration.

We can simplify this by making the object its own iterator, with the `Symbol.iterator`
method returning `this` and `next` working directly on the object's properties. The
downside of this approach is that two loops cannot iterate over the object simultaneously,
since they would share the same `current` value and interfere with each other. In the
separate iterator version this does not happen because each loop creates its own iterator
object. However, having two parallel loops is rarely needed, making the simplified version
acceptable in most cases, as shown in **ex186**.

## Exercise 189

*Iterables* are objects that implement the `Symbol.iterator` method, as we have already
seen. *Array-like objects* are objects that have indices and a `length` property, making
them look like arrays. Strings are both iterable and array-like, but an iterable may not
be array-like and vice versa. Iterables and array-like objects are generally not real
arrays since they do not have methods like `push` and `pop`, which makes them inconvenient
to work with as if they were arrays, as shown in **ex189**.

## Exercise 190

The universal method `Array.from` takes an iterable or array-like object and converts it
into a real `Array`, allowing us to use array methods on the new object. It first checks
whether the object is iterable or array-like, then creates a new array with all its
items. The syntax is: `Array.from(obj[, mapFn, thisArg])`. The optional second argument
`mapFn` is a function applied to each element before adding it to the array, and
`thisArg` allows us to set `this` for that function.

Unlike `split`, `Array.from` relies on the iterable nature of the string, so just like
`for..of`, it handles 2-byte characters such as emojis and other special characters
correctly. With `Array.from` we can even build a `slice` function that copies a portion
of a string while correctly recognizing special characters — first converting the string
into an array, then using `slice` on the array indices, and finally joining it back into
a string with `join`, as shown in **ex190**.
