# Methods of Primitives

## Exercise 143

JavaScript allows us to work with primitive types as if they were objects. The distinctions
between primitives and objects are as follows: there are seven primitive types — `string`,
`number`, `bigint`, `boolean`, `symbol`, `null`, and `undefined` — while an object can
store multiple values as properties and can be created with `{}`, for example:
`{name: "John", age: 30}`. Functions are also a type of object, and an object can store
functions as one of its properties. There are many pre-defined objects, such as those that
work with dates, errors, and HTML elements, each with different properties and methods.
However, because of all this, an object is heavier than a primitive, as shown in
**ex143**.

## Exercise 144

We can do many things with primitive types by accessing them through a method, but
primitives must remain as lightweight as possible. It may seem contradictory, but
primitive values remain primitives — a single value, as intended. The language allows
access to methods and properties of strings, numbers, booleans, and symbols. To make this
work, an *object wrapper* that provides the extra functionality is created and then
immediately destroyed. Object wrappers are different for each type and are called `String`,
`Number`, `Boolean`, `Symbol`, and `BigInt`, each providing a different set of methods.
For example, the string method `.toUpperCase()` returns the string in uppercase.

Here is what happens when `str.toUpperCase()` is called:

1. `str` is a primitive string.
2. At the moment its property is accessed, a special object is created that knows the
value of the string and has useful methods such as `.toUpperCase()`.
3. That method runs and returns a new string.
4. The special object is destroyed, leaving the primitive `str` as it was.

This is how primitive types provide methods while remaining lightweight. Numbers also have
their own methods, such as `.toFixed(n)`, which rounds the number to the specified
precision, as shown in **ex144**.

## Exercise 145

In JavaScript it is possible to explicitly create *object wrappers* for primitive types
using constructor syntax, but it is not recommended. This is because objects are always
treated as truthy, regardless of whether the value inside is zero or any other falsy
value — it will always be an object and therefore always true.

On the other hand, it is perfectly fine to use the same functions `String`, `Number`, and
`Boolean` without `new`, as they correctly convert values to their respective primitive
types. It is also worth knowing that the special primitives `null` and `undefined` are
exceptions — they have no corresponding object wrappers and provide no methods, as shown
in **ex145**.
