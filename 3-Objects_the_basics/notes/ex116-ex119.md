# JavaScript Object References and Copying

## Exercise 116

When we copy a primitive value, we get two distinct and independent variables storing the same value.

Objects work differently. A variable assigned to an object stores not the object itself, but its address, that is, a reference to it.

When an object variable is copied, the reference is copied, but the object itself is **not** duplicated. As a result, we can access either variable and modify the same underlying object.

## Exercise 117

If we compare two variables that reference the same object, they are considered equal.

However, if we create two different objects with identical contents, they are **not** considered equal, because they reference different objects, even though their values are the same.

In comparisons such as `obj1 < obj2` or `obj1 == 5`, objects are converted to primitive values. These kinds of comparisons are rarely needed and are usually the result of a programming mistake.

Something that may be unexpected is that an object declared with `const` **can still be modified**. Its properties may be changed because `const` only prevents reassignment of the variable itself, not modification of the object it references.

## Exercise 118

If we need to duplicate an object, we can iterate over its properties and copy them one by one at the primitive level, using a `for` loop to assign each property to a new clone object.

We can also use the `Object.assign(dest, ...sources)` method.

Here, `dest` is the target object, and the remaining arguments are a list of source objects.

It copies all properties from the source objects into `dest` and then returns `dest`.

If a property with the same name already exists, it is overwritten.

`Object.assign()` can also be used to perform a simple object clone.

## Exercise 119

Object properties can themselves reference other objects.

In this case, a simple assignment such as `clone.sizes = user.sizes` is not enough, because `user.sizes` is an object and will be copied **by reference**.

To make `user` and `clone` truly independent objects, we must use a cloning process that examines each value in `user[key]`. If a value is an object, its structure must also be duplicated.

This process is called **deep cloning** or **structured cloning**.

JavaScript provides the `structuredClone(object)` method, which creates a deep copy of `object`, including all nested properties.

This method also supports **circular references**, where an object contains a property that references the object itself.

However, `structuredClone()` has some limitations. For example, it cannot clone objects that contain function properties.

For more complex cases, we may need to combine different cloning techniques, write custom cloning logic, or use an existing implementation such as `_.cloneDeep(obj)` from the **Lodash** library.
