# Object to Primitive Conversion

In JavaScript we cannot use operators directly with objects, such as `obj1 + obj2` or
`obj1 - obj2`. In these cases, objects are converted into primitive types — meaning the
result of an operation between two objects cannot be another object. When this happens, it
is usually due to a programming mistake, with rare exceptions. What we will learn here is
to understand what happens when such an operation occurs accidentally, and to discover
that there are cases where these operations are intentional and work correctly.

---

## Conversion Rules

A few rules apply to object conversion:

- There is no boolean conversion. All objects are `true` in a boolean context.
- There are only numeric and string conversions.
- Numeric conversion occurs when we subtract objects or apply mathematical functions. For
example, `Date` objects can be subtracted, and the result of `date1 - date2` is the time
difference between two dates.
- String conversion generally occurs when we output an object with `alert(obj)` and in
similar contexts.

## Exercise 139

There are three variations of type conversion:

- **String** — occurs when an operation on an object expects a string result, such as
`alert`.
- **Number** — occurs when mathematical operations are performed on an object.
- **Default** — occurs when JavaScript is not sure which type to convert to, such as in
binary addition operations that work with both strings and numbers, or when an object is
compared with a string, number, or symbol.

The greater-than and less-than operators can also work with both strings and numbers, but
they convert to `"number"` rather than `"default"`.

When performing a conversion, JavaScript tries to find and call three object methods in
order:

1. `obj[Symbol.toPrimitive](hint)` — the method with the symbolic key
`Symbol.toPrimitive`, if it exists.
2. If the hint is `"string"`, it tries to call `obj.toString()` or `obj.valueOf()`,
whichever is available.
3. If the hint is `"number"` or `"default"`, it tries to call `obj.valueOf()` or
`obj.toString()`, whichever is available.

## Exercise 141

In the absence of `Symbol.toPrimitive`, JavaScript tries to find the `toString` and
`valueOf` methods:

- For the `"string"` hint: `toString` takes priority. If it returns an object instead of
a primitive, or does not exist, `valueOf` is used instead.
- For other hints: `valueOf` takes priority. If it returns an object or does not exist,
`toString` is used instead.

If `toString` or `valueOf` return an object, it is ignored as if the method did not exist.
By default, an object has the value `"[object Object]"` for `toString`, and `valueOf`
returns the object itself.

We can use a combination of `toString` and `valueOf` instead of `Symbol.toPrimitive` to
achieve the same behavior. In the absence of both `Symbol.toPrimitive` and `valueOf`, the
`toString` method can handle all primitive conversions, as shown in **ex141**.

Many operators and functions perform type conversions — for example, multiplication

calculation steps:

1. The object is converted to a primitive type using the rules described above.
2. If necessary for further calculations, the primitive value is also converted.

## Additional

Conversion methods do not necessarily return the suggested primitive type, since there is
no enforcement over whether `toString` returns a string or `Symbol.toPrimitive` returns a
number. The only requirement is that these methods return a primitive type, not an object.

For historical reasons, if `toString` or `valueOf` return an object, no error is thrown —
the value is simply ignored as if the method did not exist. This is because JavaScript did
not have a well-defined error concept in its early days.

In contrast, `Symbol.toPrimitive` is stricter and must return a primitive type, otherwise
an error will occur.
