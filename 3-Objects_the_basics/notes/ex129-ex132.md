# JavaScript Optional Chaining '?.'

## Exercise 129

Optional chaining `?.` is a safe way to access nested object properties, even if an
intermediate property does not exist. For example, if we try to access a nested object
that does not exist, JavaScript will always throw an error. In many cases however, it is
preferable to get `undefined` or `null` as a result instead.

To solve this we would have to check the value using `if` or the `?` operator, but that
quickly becomes messy, especially for more deeply nested properties where we would have to
write very long and repetitive code. A slightly better approach is to use the `&&`
operator, which shortens the code a bit, but it still ends up being very large and
repetitive, as shown in **ex129**.

## Exercise 130

Optional chaining stops the evaluation if the value before `?.` is `null` or `undefined`,
and returns `undefined`. This allows the same code we were writing to be expressed in a
much simpler way.

The syntax makes only the value immediately before `?.` optional, but not the subsequent
ones. For example, in `user?.address.street.name`, only `user` is optional — if it is
`null` or `undefined`, the expression returns `undefined`. The other properties are still
accessed in the usual way, so if we want any of them to also be optional, we need to
replace their `.` with `?.` as well.

Do not overuse optional chaining. Use it only when a specific property is genuinely
optional — otherwise we may end up hiding an error that should not happen on a required
property.

If the variable itself does not exist, such as `user` in `user?.anything`, an error will
still be thrown, since optional chaining only works with declared variables, as shown in
**ex130**.

## Exercise 131

The `?.` operator immediately stops the evaluation if the left side does not exist.
Therefore, if there are other function calls or operations to the right of `?.`, they will

## Exercise 132

Optional chaining is a special syntactic construct that also works with functions and
brackets:

- `?.()` — used to call a function that may not exist. If the function exists it is called
normally, otherwise the evaluation stops without throwing an error.
- `?.[]` — used to access properties that may or may not exist.
- `?.` with `delete` — used to delete an object or a nested property only if the optional
part exists.

One thing we cannot do is use `?.` on the left side of an assignment, since optional
chaining has no use for writing, as shown in **ex132**.
