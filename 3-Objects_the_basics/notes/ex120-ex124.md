## Exercise 122

For a method to access the information stored inside an object, we use the `this` keyword.
`this` refers to the object before the dot.

Technically, we can access the object without `this` by using its variable name directly,
since it acts as a kind of substitution. However, this approach is unreliable, because it
forces the properties to be tied to a single object. If we have multiple objects
referencing the same values and we do not use `this`, overwriting the object variable will
cause an error, as shown in **ex122**.

## Exercise 123

In JavaScript, `this` can be used in any function, even if it is not a method, since the
value of `this` is evaluated at runtime. This means we can declare a function with `this`
first and then assign that same function as a method to different objects.

We can even call the function without any object. In that case, in strict mode `this` will
be `undefined`, and attempting to access `this.name` will throw an error. In non-strict
mode, the value of `this` will be the *global object*, but this kind of occurrence is
generally a programming mistake.

Because `this` in JavaScript is "free" and its value is evaluated at the time of the
call, we can reuse functions across different objects — something that does not happen in
other languages, where methods always have a `this` that references their own object. In
JavaScript, this creates more flexibility, but also more room for mistakes, as shown in
**ex123**.
