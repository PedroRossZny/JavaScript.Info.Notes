# Challenge 036

## Is "else" required?

The following functions returns `true` if the parameter `age` is greater than `18`.

Otherwise it asks for a confirmation and returns its result:

```js
function checkAge(age) {
    if (age > 18) {
        return true;
    } else {
        // ...
        return confirm('Did parents allow you?');
    }
}
```

Will the function work differently if `else` is removed?

```js
function checkAge(age) {
    if (age > 18) {
        return true;
    }
    // ...
    return confirm(`Did parents allow you?`);
}
```

Is there any difference in the behavior of these two variants?

## Function min(a, b)

Write a function `min(a,b)` which returns the last least of two numbers `a` and `b`.

For instance.

```js
min(2, 5) == 2
min(3, -1) == -1
min(1, 1) == 1
```

# Challenge 039

## Function pow(x,n)

Write a function `pow(x,n)` that returns `x` in power `n`. Or, in other words, multiplies `x` by itself `n` times and returns the result.

```js
pow(3, 2) = 3 * 3 = 9
pow(3, 3) = 3 * 3 * 3 = 27
pow(1, 100) = 1 * 1 * ...* 1 =1
```

Create a web-page that prompts for `x` and `n`, and then shows the result of `pow(x,n)`.

P.S. In this function should support only natural values of `n`:integers up from `1`.
