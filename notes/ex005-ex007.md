# JavaScript Code Structure

## Exercise 005

We separate the statements we have seen so far using semicolons. We can write more than one statement on the same line by using this separator, such as placing two `alert` statements on a single line. Normally, statements are written on separate lines to improve readability.

In most cases, the semicolon can be omitted because JavaScript interprets a line break as an implicit semicolon. However, there are situations where JavaScript does not do this, usually because an expression has not been completed and continues on the following lines. As shown in **ex005**.

## Exercise 007

To create single-line comments, use `//` before the text on the corresponding line. It can occupy an entire line or appear after a statement.

For multi-line comments, use `/*` at the beginning and `*/` at the end. The content inside comments is ignored, so code can be placed within them.

Always consider using comments to improve the readability of your code. As shown in **ex007**.
