# JavaScript Conditional Branching: if, '?'

## Exercise 041

The `if(...)` statement evaluates a condition inside the parentheses. If the result is `true`, a block of code is executed.

For a simple check, the entire statement can be written on a single line. However, if multiple statements need to be executed, they can be enclosed within curly braces. Using braces is generally recommended, even for single statements, as it improves readability.

The `if(...)` statement evaluates an expression and converts the result to a boolean value.

Recall the conversion rules:

* `0`
* `""`
* `null`
* `undefined`
* `NaN`

are converted to `false`.

All other values become `true`. Such values are commonly called *truthy* values.

As a result, code such as `if (0)` will never be executed.

It is also possible to pass a previously evaluated boolean value to an `if` statement. As shown in **ex041**.

## Exercise 044

Sometimes we need to assign a value to a variable based on a condition. The conditional operator allows us to do this in a short and simple way.

It is represented by `?` and is also known as the *ternary operator* because it works with three operands.

The syntax is:

`let result = condition ? value1 : value2;`

The condition is evaluated. If it is `true`, `value1` is returned; otherwise, `value2` is returned.

The parentheses around the condition can be omitted because the question mark operator has relatively low precedence. However, keeping the parentheses is usually recommended because it makes the code easier to read. As shown in **ex044**.
