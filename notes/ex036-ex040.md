# JavaScript Comparisons

## Exercise 036

There are comparison operators: greater than `>`, less than `<`, greater than or equal to `>=`, less than or equal to `<=`, equality `==`, and inequality `!=`.

Note that the double equals operator `==` performs an equality test, while a single equals sign `=` is used for assignment.

All comparison operators return a boolean value, either `true` or `false`. The result of a comparison can be assigned to a variable just like any other value. As shown in **ex036**.

## Exercise 037

To determine whether one string is greater than another, JavaScript follows a *dictionary* or *lexicographical* order. In other words, strings are compared character by character.

The algorithm works as follows:

1. Compare the first character of both strings.
2. If the first character of one string is greater or smaller than the first character of the other string, the comparison result is determined immediately.
3. Otherwise, if the first characters are equal, the comparison continues with the next characters until a difference is found or both strings end.
4. If all compared characters are equal, the longer string is considered greater.

The comparison is not based on a real dictionary order but on Unicode values. This means that uppercase and lowercase letters matter. Lowercase letters are considered greater than their uppercase counterparts because they have higher Unicode values. As shown in **ex037**.

## Exercise 038

When comparing values of different types, JavaScript converts them to numbers.

For boolean values, `true` becomes `1` and `false` becomes `0`.

Be careful: an equality comparison may convert values using numeric conversion rules, while an explicit `Boolean()` conversion follows a different set of rules. As shown in **ex038**.

## Exercise 039

A regular equality check `==` cannot distinguish `0` from `false`, nor from an empty string, because the operands are converted before the comparison is performed.

A strict equality operator `===`, however, checks equality without performing type conversion. Therefore, comparing `0` with `false` using `===` returns `false`.

There is also a strict inequality operator `!==`, which is the strict counterpart of `!=` and performs inequality checks without type conversion. As shown in **ex039**.

## Exercise 040

When performing a strict equality comparison `===` between `null` and `undefined`, they are considered different because they belong to different types.

However, in a non-strict comparison `==`, they are considered equal.

When using other comparison operators such as `<`, `>`, `<=`, or `>=`, JavaScript converts the values. `null` is converted to `0`, while `undefined` is converted to `NaN`.

This behavior can be confusing. Although `null == undefined` returns `true`, neither value is considered equal to any other value. For example, neither `null == 0` nor `undefined == 0` returns `true`.

On the other hand, `null >= 0` returns `true` because `null` is converted to `0` during the comparison. The same does not happen with `undefined`, since it is converted to `NaN`, and comparisons involving `NaN` always return `false`.

As a result, an equality comparison involving `undefined` returns `true` only when compared with `null` using `==`.

It is generally recommended to avoid comparisons involving `null` or `undefined`, except when using strict equality `===` or strict inequality `!==` with a clear understanding of the intended behavior. In particular, avoid using `>=`, `>`, `<`, or `<=` with `null` or `undefined` unless you are completely certain of the result. As shown in **ex040**.
