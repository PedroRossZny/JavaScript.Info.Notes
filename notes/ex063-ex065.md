# JavaScript The `switch` Statement

## Exercise 063

A `switch` statement can be used as an alternative to multiple `if` checks.

A `switch` contains one or more `case` clauses and an optional `default` clause.

The value provided to `switch` is compared against each `case` value using strict equality (`===`).

If a match is found, the code associated with that `case` is executed until the nearest `break` statement is reached.

If no matching `case` is found, the code inside the `default` clause is executed, if one exists.

In other words, if a `case` does not match, execution continues to the next `case` or to `default`.

If a `break` statement is omitted, execution continues from the first matching `case` through all subsequent cases, regardless of their conditions. This behavior is known as *fall-through*.

Any expression can be used in a `switch` or `case`, since both support arbitrary expressions. For example:

```js id="zt2r9r"
switch (+a)
```

or

```js id="4ckhz0"
case b + 1
```

## Exercise 064

Multiple `case` clauses can be grouped together.

This is done by placing one or more `case` clauses without code or a `break` statement above another `case` that contains the actual code and `break`.

As a result, all grouped cases execute the same block of code when any of them matches.

This behavior is a consequence of the fall-through mechanism caused by the absence of a `break` statement.

## Exercise 065

The type of the values used in `switch` and `case` statements is very important.

Because `switch` uses strict equality (`===`) for comparisons, both the value and the type must match.

For example, the number `3` and the string `"3"` are considered different values and will not match the same `case`.

Always ensure that the values being compared have compatible types.
