# JavaScript The Modern Mode, "use strict"

## Exercise 008

`"use strict"` or `'use strict'` is placed on the first line of the code to enable modern mode. JavaScript often allows many "mistakes" to pass by trying to interpret the code in the best possible way and compensate for potential issues to avoid errors.

When `strict` mode is enabled, this behavior does not occur. Errors are exposed more clearly, and JavaScript becomes less permissive. As shown in **ex008**.

## Exercise 009

If you do not place `"use strict"` at the beginning of your code, strict mode may not be enabled. Therefore, make sure it is declared correctly. As shown in **ex009**.

## Additional

It is recommended to use strict mode. However, JavaScript supports *classes* and *functions* where `"use strict"` is enabled automatically.
