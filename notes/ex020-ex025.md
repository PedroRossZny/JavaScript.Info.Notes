# JavaScript Data Types

## Exercise 020

JavaScript is a language known as **dynamically typed**, meaning that data types exist, but variables are not bound to a specific type. The `number` type represents both integers and floating-point values. There are many operations that can be performed with numbers, such as `*` (multiplication), `/` (division), `+` (addition), `-` (subtraction), and so on.

In addition to regular numbers, JavaScript provides special numeric values: `Infinity`, `-Infinity`, and `NaN`. `Infinity` represents mathematical infinity, while `NaN` represents a mathematical error. `NaN` is persistent: any operation involving `NaN` will return `NaN`, except for `NaN ** 0`, which evaluates to `1`.

Numeric calculations never produce a fatal error. In the worst-case scenario, the result will be `NaN`, as shown in **Exercise 020**.

## Exercise 021

In JavaScript, the `number` type cannot safely represent integer values greater than `9007199254740991` or less than `-9007199254740991`, because not all digits fit within the fixed 64-bit storage format.

For this reason, JavaScript provides `BigInt`, a type used to represent integers of arbitrary length. A `BigInt` value is created by appending an `n` to the end of a number literal, as shown in **Exercise 021**.

## Exercise 022

JavaScript supports three ways to create strings: double quotes `"Hello"`, single quotes `'Hello'`, and backticks `` `Hello` ``.

Double and single quotes provide essentially the same functionality. However, backticks offer extended capabilities through **template literals**, allowing variables and expressions to be embedded directly inside a string using `${...}`.

The expression inside `${...}` is evaluated, and its result becomes part of the string. It can contain a variable or even an arithmetic expression, as shown in **Exercise 022**.

## Exercise 024

In JavaScript, `null` is a special value that represents "nothing", "empty", or "unknown value".

The special value `undefined` means that "no value has been assigned".

Generally, `null` is used to intentionally assign an empty value, while `undefined` is reserved as the default initial value for unassigned variables and properties, as shown in **Exercise 024**.

## Exercise 025

The `typeof` operator returns the type of its operand. The expression `typeof x` returns a string containing the name of the type.

`Math` is a built-in object that provides mathematical operations.

The fact that `typeof null` returns `"object"` is an officially recognized mistake that has been preserved for compatibility reasons. `null` is not an object; it is a special value with its own type.

Functions belong to the `object` type, but `typeof` treats them differently and returns `"function"` instead. This behavior is technically inconsistent but is often convenient in practice, as shown in **Exercise 025**.

## Additional

Some programming languages have a special type for a single character. In C and Java, this type is called `char`. JavaScript, however, has only one text type: `string`.

`typeof x` and `typeof(x)` are exactly the same. This does not mean that `typeof` is a function—it is an operator. The parentheses work the same way they do in a mathematical expression, such as `(2 + 2)`.
