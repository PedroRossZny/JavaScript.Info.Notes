# JavaScript Arrow Functions, the Basics

## Exercise 079

There is another simple and concise way to create functions in JavaScript called *arrow functions*.

Arrow functions are often more convenient than function expressions and can be seen as a shorter version of them.

An arrow function can be assigned to a variable and consists of a list of parameters followed by the `=>` operator. The expression on the right side of `=>` is evaluated, and its result is returned automatically.

If there is only one parameter, the parentheses may be omitted.

If there are no parameters, the parentheses must still be present, but they remain empty.

Arrow functions can be used in the same situations as function expressions, such as creating functions dynamically or passing functions as arguments.

They are especially convenient for writing short, single-line actions.

## Exercise 080

When a function requires multiple statements or more complex logic, curly braces can be placed after the `=>`.

In this form, the code inside the braces becomes the function body.

Unlike single-expression arrow functions, a function body enclosed in braces does not return a value automatically.

To return a value, an explicit `return` statement must be used, just as in a regular function.
