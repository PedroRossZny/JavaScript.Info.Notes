# JavaScript Loops: `while` and `for`

## Exercise 055

In programming, we often need to repeat actions.

The first loop type is `while`. It starts with a condition that is evaluated before each iteration. If the condition is `true`, the loop continues and the code inside its body is executed.

Each execution of the loop body is called an *iteration*.

If no iteration causes the condition to become `false`, the loop will continue indefinitely, creating an infinite loop.

Any expression or variable can be used as a loop condition because the condition is evaluated and converted to a boolean value.

If the loop body contains only a single statement, the curly braces `{...}` may be omitted.

## Exercise 056

The condition check can also be placed after the loop body by using `do...while`.

With this structure, the loop body is executed first, and only then is the condition evaluated. As long as the condition remains `true`, the loop continues to execute.

Normally, the regular `while` loop is preferred. However, if the loop body must be executed at least once regardless of the condition, `do...while` is the correct choice.

## Exercise 057

The `for` loop is more complex than `while`, but it is also the most commonly used loop.

A typical `for` loop consists of four parts:

* **Begin**: `let i = 0`
  Executed once when entering the loop.

* **Condition**: `i < 3`
  Checked before each iteration. If it becomes `false`, the loop stops.

* **Body**: `alert(i)`
  Executed repeatedly while the condition remains `true`.

* **Step**: `i++`
  Executed after the body during each iteration.

The begin section runs only once. After that, the cycle follows this order:

`condition → body → step`

This sequence repeats until the condition becomes `false`.

## Exercise 059

Any part of a `for` loop can be omitted.

The begin section may be omitted if no variable needs to be declared or initialized.

The step section may also be omitted, leaving only the condition. In this case, the loop behaves similarly to a `while` loop.

It is even possible to omit all three sections, creating an infinite loop:

```js
for (;;) {
  // loop body
}
```

However, the two semicolons must always remain. Omitting them results in a syntax error.

## Exercise 060

Normally, a loop ends when its condition becomes `false`.

However, we can force the loop to stop at any time using the special `break` statement.

The combination of an *infinite loop + `break` when needed* is useful in situations where the loop condition must be checked in the middle of the loop body or at multiple points within it.

## Exercise 061

The `continue` statement is a lighter version of `break`.

Instead of terminating the entire loop, it stops only the current iteration and immediately proceeds to the next one, if the loop condition allows it.

This statement can help reduce nested code structures and improve readability.

The `break` and `continue` statements cannot be used with the ternary operator `? :`. They are not allowed in that context.

This is another reason why `if` statements should be preferred when control-flow statements such as `break` or `continue` are needed.

## Exercise 062

Sometimes we need to exit multiple nested loops at the same time.

A regular `break` only terminates the innermost loop, so JavaScript provides *labels* for such situations.

A label is an identifier followed by a colon and placed before a loop.

We can then write `break labelName`, and JavaScript will terminate the loop associated with that label.

Labels may also be placed on a separate line for readability.

Similarly, `continue labelName` can be used to continue the next iteration of a labeled loop.

Labels do not allow arbitrary jumps within the code. They only work together with `break` and `continue`, and these statements must appear inside a code block associated with the corresponding label.
