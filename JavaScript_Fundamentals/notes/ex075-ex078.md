# JavaScript Function Expressions

## Exercise 075

In JavaScript, a function is a special type of value.

The syntax we used previously is called a *function declaration*. However, there is another way to create functions called a *function expression*.

A function expression allows us to create a function within any expression. For example, a function can be assigned as the value of a variable.

Unlike function declarations, omitting the function name is allowed in function expressions.

## Exercise 076

Regardless of how a function is created, it remains a value.

In JavaScript, if we refer to a function only by its name, without parentheses, we are referring to the function itself as a value.

A function is a value, but it is a special kind of value. When we add parentheses, we execute the code contained within the function. Even so, it remains a value and can be manipulated like other values.

For example, a function can be copied to another variable. In that case, the new variable also refers to the same function.

Keep in mind that function expressions usually end with a semicolon because they are part of an expression, such as an assignment statement.

## Exercise 077

Functions can be passed as arguments to other functions.

They are often used as *callback functions*, which are functions that are executed in response to a particular event, condition, or action.

A callback can be passed by referring to an existing function's name or by declaring a function directly within the argument list.

Functions declared directly as arguments usually do not have names and are therefore called *anonymous functions*.

Anonymous functions are not accessible outside the context in which they were created.

Regular values such as strings and numbers represent *data*, while functions can be viewed as values that represent *actions*.

## Exercise 078

A *function declaration* is a function defined as a separate statement in the main code flow.

A *function expression* is a function created inside an expression or another syntactic construct.

A function expression is created only when execution reaches that part of the code and can only be used after it has been created.

A function declaration behaves differently. It can be called before its definition appears in the code.

This is possible because, before executing a script, JavaScript scans for function declarations and creates them during an initial preparation phase. This behavior is commonly known as *hoisting*.

Because of hoisting, function declarations are available throughout their scope before the code reaches their actual location.

Function expressions do not have this behavior.

Another important characteristic of function declarations is their block scope behavior in strict mode.

When a function declaration appears inside a block, it is visible throughout that block but not outside of it.

To make a function available outside the block, a common solution is to use a function expression and assign it to a variable declared in an outer scope. This allows the function to be called even after execution leaves the block.

This pattern can sometimes be simplified further using the conditional operator `?` together with function expressions.

Function declarations generally provide greater flexibility in code organization and are often easier to read, making them the preferred choice in many situations.

However, there are cases where function expressions are more appropriate, such as when a function needs to be created conditionally or assigned dynamically.
