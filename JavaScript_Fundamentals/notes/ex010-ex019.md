## Exercise 012

We can declare multiple variables in several ways. One option is to declare them on a single line, using commas to separate each variable. This approach is shorter but generally not recommended.

The easiest format to read is declaring each variable on its own line and ending each declaration with a semicolon. Some developers also use a multiline style where commas are placed at the end of each declaration and `let` is used only for the first one, with the semicolon appearing on the last line. Others do the opposite and place the commas at the beginning of declarations that are not the first. As shown in **ex012**.

## Exercise 017

A variable name may contain letters, digits, or the symbols `$` and `_`, but the first character cannot be a digit.

When a variable name contains multiple words, they are usually written together, with the first letter of each word capitalized except for the first word, which starts with a lowercase letter. For example: `myVeryLongName`.

The symbols `$` and `_` are treated as regular characters, just like letters, and have no special meaning.

A variable named `apple` and another named `APPLE` are considered two different variables.

Non-Latin characters are allowed but are generally not recommended. It is usually better to use English names.

There is also a list of reserved names used by the language itself, and these cannot be used as variable names. As shown in **ex017**.

## Exercise 019

Instead of `let`, we can also use `const`, which declares an immutable constant variable whose value cannot be reassigned.

A common practice is to use constants as aliases for values that are difficult to remember. Such constants are usually named using uppercase letters and underscores.

We should use this naming convention only when the constant is known before execution. It is generally not used when the constant is calculated at runtime. As shown in **ex019**.

## Additional

A variable name should have a clear and obvious meaning.

* Use readable names such as `userName` or `shoppingCart`.
* Do not use abbreviations or short names such as `a`, `b`, or `c`.
* Make names as descriptive and concise as possible. Poor examples include `data` and `value`, since such names do not convey meaningful information.
* Be consistent with the terminology used by your team and throughout your code. If a visitor to the site is called a "user", then related variables should be named `currentUser` and `newUser` instead of `currentVisitor` or `newManInTown`.
* Always try to create new variables when appropriate. Avoid reusing the same variable to store unrelated values.
